# Инструмент для сверки заказов и расчета выручки

Скрипт для автоматической сверки данных между выгрузкой продаж с маркетплейса и реестром заказов, с расчетом ключевых бизнес-показателей.

## 📋 Оглавление

- [Описание](#описание)
- [Установка](#установка)

## Описание

Инструмент решает следующие задачи:

1. **Загрузка и валидация данных** из Excel-файлов
2. **Расчет выручки** по каждому товару (SKU) с учетом статусов заказов
3. **Сверка данных** между выгрузкой продаж и реестром заказов
4. **Расчет ключевых показателей**:
   - Общая выручка
   - Топ-5 товаров по обороту
   - Количество и сумма возвратов
   - Доля возвратов
5. **Выявление расхождений** между источниками данных

## Установка

### Требования
- Python 3.8 или выше
- pip (менеджер пакетов Python)

### Шаги установки

```bash
# Клонировать репозиторий
git clone https://github.com/yanfeofanov/test_app_orders.git
cd test_app_orders

# Установить зависимости
pip install -r requirements.txt

# Запуск приложения 
1. Базовый запуск 
python3 main.py -s orders.xlsx -r "Реестр заказов (для сверки).xlsx"
2. Запуск c сохранением результатов в файл
python3 main.py -s orders.xlsx -r "Реестр заказов (для сверки).xlsx" -o report.txt
3. Запуск с подробным логированием
python3 main.py -s orders.xlsx -r "Реестр заказов (для сверки).xlsx" -v
4. Запуск тестов 
pytest test_order_processing.py -v
```
### Как использовал ИИ-ассистента и что перепроверял вручную;
В данном контексте, опишу своими словами и пониманием по ошибкам и багам в коде, эта часть revenue[sku] = amount - исправление нужно, но ИИ делает генерацию быстрее, поэтому синтаксический сахар был такой if sku in revenue: revenue[sku] += amount else: revenue[sku] = amount ну и дальше проработка исключений, логирование было написано ручками, но для ускорения процесса так же использовал помощника, так же баги по части return resp.json()["status"] - где если упадет а оно упадет то скрипт поломается, соответственно помощник предложил оптимальный вариант исправления использовать декоратор (доп. функционал) с задержкой по выполнению запросов, расхождение по заказам между двух файлах, сравнение сумм и количество, что предложил помощник это нахождение уникальных позиций через агригацию через groupby, находим расхождение в наборе заказов и далее сравниваем суммы и количества через функцию которую пришлось перепроверять что-то руками поправить чтобы считало правильно, так же в целом работа с множествами а не просто хардкорными данными и структурированные результаты с использованем map - он же словарь в питоне, помог так же с автотестами потому что, мокать данные долго если писать ручками, смотреть структуру файлов и т.д. соответственно сгенерировать без ошибочно строки это был большой плюс (подготовка - мокирование данных), вот как мне помог ИИ агент в данном контексте задачи 

### Как бы вывел это в расписание (cron / schtasks) и что мониторил бы в проде

   0 * * * * /usr/bin/python3 - каждый час 
   */30 * * * * /usr/bin/python3 - каждые 30 минут
